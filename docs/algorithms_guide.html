1 — What Is Clustering?

Imagine you work at a supermarket and you want to group your customers by shopping behaviour. You don't already know how many groups there are or what they look like — you just want the computer to look at the data and find natural groups automatically.

That is clustering: the task of automatically dividing a pile of data points into groups (called clusters) so that points inside the same group are as similar as possible, and points in different groups are as different as possible.

Real-world analogy

Think of sorting a huge bag of mixed buttons by colour and size without any labels. You naturally put all small red buttons together, all large blue buttons together, etc. Clustering does the same thing — but with numbers instead of buttons.

Why does it matter?

Clustering is used everywhere:

  • Marketing: group customers to target them with the right offers
  • Medicine: find patient subgroups that respond similarly to a drug
  • Recommendation systems: Netflix groups users with similar tastes
  • Image compression: represent millions of pixel colours with just a few representative ones
  • Fraud detection: unusual transactions that don't fit any normal cluster

How are clusters represented?

Each cluster is represented by a centroid — the "average" point of all members in that cluster. Think of it as the centre of gravity. Every algorithm in this dashboard ultimately tries to find the best set of centroids.

2 — What Is Optimization?

Finding the perfect centroid positions for even a small dataset is computationally impossible by brute force — there are too many possible combinations. Instead, all 15 algorithms use optimization: they start with a guess, then keep improving it step by step.

Real-world analogy

Imagine you're blindfolded on a hilly landscape and you want to find the lowest valley. You can't see the whole map, so you take a step, check if you went down, then take another step. This is basically what every optimization algorithm does — explore the space cleverly and keep the best solution found so far.

What are they optimizing?

Every algorithm tries to minimize WCSS (Within-Cluster Sum of Squares) — the total "distance" from each data point to its nearest centroid. Lower WCSS means tighter, more compact clusters. Think of it as "how far is the average customer from their group's profile".

Why so many different algorithms?

Because optimization is hard! A simple greedy approach gets stuck in local optima — decent solutions that aren't the best possible. Each algorithm uses a different strategy to avoid getting stuck and explore the solution space more effectively.

3 — The Dataset

This dashboard uses the Customer Personality Analysis dataset from Kaggle — real marketing data from 2,215 customers of a retail company.

What does each customer record contain?

  • Age — derived from birth year
  • Income — annual household income (capped at 99th percentile to remove extreme outliers)
  • Total Spend — sum of money spent on wine, fruit, meat, fish, sweets, and gold
  • Num Purchases — total number of purchases across all channels
  • Education — one-hot encoded (Basic, Graduation, Master, PhD, 2n Cycle)
  • Marital Status — one-hot encoded (Single, Married, Divorced, etc.)
  • Individual spending categories — wine, fruit, meat, fish, sweets, gold
  • Purchase channels — web, catalogue, store, deals
  • Website visits — how many times they visited the website last month
After preprocessing: ~2,215 rows × 20 features. All features are scaled to have zero mean and unit variance (StandardScaler) so that income (which is in thousands) doesn't dominate age (which is in tens).

What is "k"?

k is the number of clusters you want the algorithm to find. Setting k=4 means "please divide customers into 4 groups". Every algorithm needs to know k upfront — it's the one thing you must decide yourself.

4 — Classical Algorithms

These are fast, well-understood methods that have been used for decades. They serve as baselines — benchmarks that the more sophisticated CI/EC algorithms are compared against.

⭕
K-means
The classic, boring but reliable
Classical ▼
Analogy

You randomly drop 4 flags on a map of your customers. Each customer "belongs" to the nearest flag. Then you move each flag to the average location of all customers belonging to it. Repeat until the flags stop moving.

Step by step

  1. Initialize: Randomly pick k data points as starting centroids.
  2. Assign: Label each customer with the nearest centroid — this creates k clusters.
  3. Update: Move each centroid to the mean (average) position of all customers assigned to it.
  4. Repeat: Go back to step 2. Keep repeating until no customer changes cluster (or max iterations reached).

Key parameters

ParameterWhat it does
kHow many clusters to find.
max_iterMaximum number of assign-then-update cycles before giving up.
seedControls the random starting positions. Different seeds give different results.

✅ Strengths

  • Extremely fast
  • Simple to understand
  • Works well on globular clusters

❌ Weaknesses

  • Very sensitive to starting positions (bad luck = bad result)
  • Often gets stuck in local optima
  • Struggles with non-spherical or unequal-sized clusters
⭕➕
K-means++
Smarter starting positions
Classical ▼
Analogy

Same as K-means, but instead of dropping flags randomly, you drop the first flag randomly, then deliberately place the next ones as far as possible from existing flags. This spreads them out and avoids the "two flags in the same corner" problem.

Step by step

  1. Smart init: Pick the first centroid randomly. For each remaining centroid, choose a data point at random — but data points far from existing centroids have a higher probability of being chosen.
  2. Then it's just K-means: Once all centroids are placed, run the same assign → update → repeat loop as plain K-means.

Why is it better?

By spreading starting centroids evenly across the data, K-means++ dramatically reduces the chance of landing in a terrible local optimum. On average it gets much better results than plain K-means for the same speed.

✅ Strengths

  • Usually much better than plain K-means
  • Still very fast
  • Theoretically guaranteed to be within a log factor of optimal

❌ Weaknesses

  • Still only one run — still can get unlucky
  • Still assumes spherical clusters
🔗
KNN Clustering
Guilt by association
Classical ▼
Analogy

Imagine every customer is a person standing in a room. Each person reaches out and holds hands with their 10 nearest neighbours. Groups form naturally wherever chains of handholding connect people. Then you cut the weakest links to get exactly k groups.

Step by step

  1. Build a graph: For each customer, find their 10 nearest neighbours and connect them with an edge (a line). Result: a web of connections called a "K-nearest-neighbour graph".
  2. Hierarchical clustering: Use Ward agglomerative clustering on this graph. It merges the two closest groups repeatedly, bottom-up, until k groups remain.
  3. Done: No iterative refinement — the result is deterministic (same answer every time with the same data).

Key difference

This method captures the topology (shape) of the data, not just distance to a centroid. It can find elongated, curved, or irregular clusters that K-means cannot.

✅ Strengths

  • Can find non-spherical clusters
  • Completely deterministic — same result every time
  • Captures local neighbourhood structure

❌ Weaknesses

  • Slower for large datasets
  • WCSS score can be misleadingly high (centroids are not the optimization target here)
  • Doesn't refine clusters after initial assignment
🔗➕
KNN++
Best of both worlds
Classical ▼
Analogy

Same as KNN Clustering, but before building the hand-holding graph, you first use K-means++ to find good starting positions. Then you use those positions to figure out how many neighbours each person needs — adjusting the "reach" of each person based on how sparse or dense their part of the room is.

Step by step

  1. Smart seeds: Use K-means++ initialization to place k seed centroids across the data.
  2. Adaptive k: Compute how many neighbours to use based on the data density: k = max(5, n_samples / (n_clusters × 5)).
  3. Build graph & cluster: Same Ward agglomerative clustering on the KNN graph as in KNN Clustering.

✅ Strengths

  • Combines smart seeding with graph structure
  • Adaptive connectivity based on data density
  • More robust than plain KNN in practice

❌ Weaknesses

  • Still doesn't iteratively refine centroids
  • Most complex of the four classical methods

5 — CI/EC Metaheuristic Algorithms

These 11 algorithms are inspired by nature — ant colonies, bird flocks, evolution, music, political empires. They are all population-based or iterative search methods designed to explore the solution space more thoroughly than K-means and avoid getting trapped in bad local optima.

All of them ultimately produce centroids and can be compared fairly because they all minimize the same objective: WCSS.

Group A — Nature-Inspired Heuristics

🎵
Harmony Search
Jazz improvisation
CI / EC ▼
Analogy

Imagine a jazz band with a "harmony memory" — a notebook of all the best musical phrases played so far. When creating a new piece, each musician either plays a phrase from the notebook (with a chance of slightly tweaking the pitch) or plays something completely new. Over time, the notebook fills up with better and better music, and bad phrases get replaced.

Step by step

  1. Initialize Harmony Memory (HM): Create pop_size random centroid configurations. This is the "notebook".
  2. Generate a new harmony: For each centroid dimension, flip a coin (probability hmcr=0.9). If heads: copy a value from the HM. If tails: generate a random new value.
  3. Pitch adjustment: If you copied from the HM, with probability par=0.3, nudge the value slightly (add a small random amount). The nudge size (bandwidth) shrinks over time so you go from exploring to fine-tuning.
  4. Evaluate & update: Compute WCSS for the new harmony. If it's better than the worst one in the HM, replace it.
  5. Repeat for max_iter iterations.

Key parameters

ParameterWhat it does
hmcr=0.9Harmony Memory Consideration Rate — 90% chance of using the notebook rather than going random.
par=0.3Pitch Adjustment Rate — 30% chance of tweaking a notebook value.
bw_init=0.02Starting bandwidth (tweak size). Shrinks over time to fine-tune.

✅ Strengths

  • Good balance of exploration and exploitation
  • Simple tuning: HMCR and PAR are intuitive
  • Memory mechanism preserves best-so-far solutions

❌ Weaknesses

  • Performance sensitive to bandwidth decay schedule
  • Can converge prematurely if bandwidth shrinks too fast
🔥
Extremal Optimization (EO)
Fire the worst employee every day
CI / EC ▼
Analogy

You have a team of k workers (centroids). Every day you measure each worker's performance (contribution to WCSS). The worst performer is fired immediately and replaced with a random new hire. Then everyone settles into their best positions for this team. Repeat until no improvement.

Step by step

  1. Initialize: Start with k random centroids and do one Lloyd's refinement step.
  2. Score each centroid: For each centroid, measure the WCSS it contributes (sum of squared distances for all points in its cluster).
  3. Kill the worst: Find the centroid with the highest cost. Replace it with a random data point from the dataset.
  4. Refine: Run one Lloyd's iteration (reassign all points to nearest centroid, update centroid means).
  5. Track best: If this new solution is better than the best seen so far, save it.
  6. Repeat for max_iter iterations.

Key insight

This is a single-solution algorithm — it maintains one current solution at a time, unlike population-based methods. By always replacing the worst component, it drives the system toward a better configuration through a chain of local improvements.

✅ Strengths

  • Very simple — almost no parameters
  • Aggressive improvement: directly attacks the worst element

❌ Weaknesses

  • Deterministic worst-selection may miss better solutions
  • No population diversity — can get stuck
🎲
τ-Extremal Optimization (τ-EO)
Fire with a lottery — not always the worst
CI / EC ▼
Analogy

Same "fire and replace" idea as EO, but instead of always firing the absolute worst worker, you hold a lottery. Bad performers have a much higher chance of being selected, but even mediocre or decent workers can occasionally get fired. This randomness helps escape local traps that plain EO gets stuck in.

Step by step

  1. Initialize: Same as EO.
  2. Rank centroids: Sort by cost from worst (rank 1) to best (rank k).
  3. Weighted lottery: Assign probability to each centroid: P(rank) = rank^(-τ) / sum. With τ=1.5, the worst centroid has ~5× higher chance of being picked than the best — but everyone has a chance.
  4. Replace & refine: Same as EO — replace chosen centroid with random data point, run Lloyd's step.
  5. Repeat for max_iter iterations.

What is τ?

τ (tau) controls the selection pressure. Higher τ → more deterministic (almost always fires the worst). Lower τ → more random. τ=1.5 is a sweet spot that works well in practice.

✅ Strengths

  • Escapes local optima better than plain EO
  • The probabilistic selection naturally balances exploration vs exploitation

❌ Weaknesses

  • τ must be tuned for best results
  • Still single-solution — no population diversity
🏛️
Imperialist Competitive Algorithm (ICA)
Empires conquering colonies
CI / EC ▼
Analogy

History class: 5 powerful empires each control a bunch of colonies. Colonies slowly drift toward their empire's capital (assimilation). Occasionally a rebel colony revolts and teleports to a random location (revolution). Weaker empires lose colonies to stronger ones. Eventually one empire dominates — that's your best solution.

Step by step

  1. Initialize population: Create pop_size random centroid configurations.
  2. Form empires: Rank all configurations by WCSS. The 5 best become imperialists (empire capitals). The rest become colonies assigned to empires proportionally to how good the imperialist is.
  3. Assimilation: Each colony moves a random fraction of the way toward its imperialist (centroids shift closer to the empire capital).
  4. Revolution: With probability 10%, a colony teleports to a completely random position — this prevents all colonies from being identical copies.
  5. Colony competition: If a colony becomes better than its imperialist, they swap roles.
  6. Imperial competition: The weakest empire loses its worst colony to the strongest empire.
  7. Collapse: An empire with no colonies is dissolved; its imperialist becomes a colony of the strongest empire.
  8. Repeat until convergence or max iterations.

Key parameters

ParameterWhat it does
n_empires=5Number of competing empires.
assimilation_coeff=2.0How quickly colonies move toward their imperialist.
revolution_rate=0.1Probability that a colony randomly "revolts" each iteration.

✅ Strengths

  • Rich diversity — many simultaneous solutions
  • Sociopolitical metaphor keeps balance between exploitation (assimilation) and exploration (revolution)

❌ Weaknesses

  • Many moving parts — harder to debug
  • Computationally heavier than simpler methods

Group B — Ant Colony Optimization (ACO)

All four ACO variants use the same underlying framework called ACOR (Ant Colony Optimization for Continuous domains). They maintain an archive of good solutions and sample new solutions from a mixture of Gaussian distributions centred on archived solutions. The variants differ only in how they weight which archived solutions are sampled more often.

Common ACO concept: In the original ACO for graphs, ants leave pheromone trails on paths they use — the better a path, the more pheromone, and thus more ants use it. In ACOR, the "pheromone" is represented as weights given to solutions in the archive. Better solutions have higher weights and are sampled more often.
🐜
Ant System (AS)
Ants leaving trails to food
ACO ▼
Analogy

Ants leave pheromone trails to food sources. Every ant has an equal voice — no ant's trail counts more than any other's. Over time, the colony samples all known food sources equally and keeps improving the archive.

Step by step

  1. Initialize archive: Generate pop_size random centroid solutions, evaluate WCSS, sort best to worst.
  2. Assign equal weights: Every archived solution gets weight 1/pop_size. All equally influential.
  3. Sample a new solution: For each dimension of the centroid vector, randomly pick one archived solution (using the weights), then sample a new value from a Gaussian centred on that solution's value with a spread based on how spread out the archive is.
  4. Evaluate & update archive: If the new solution is better than the worst in the archive, it replaces the worst. Re-sort.
  5. Repeat for max_iter iterations.

✅ Strengths

  • Simple and well-understood
  • Good diversity — all archive solutions treated equally

❌ Weaknesses

  • May be slow to converge — wastes effort on bad solutions
  • No mechanism to focus on the best
🐜👑
Elitist Ant System (EAS)
The queen ant gets extra influence
ACO ▼
Analogy

Same as Ant System, but the best solution (the "queen ant") leaves extra pheromone. She has twice the influence of any regular ant — all other ants are drawn more toward her trail.

How it differs from AS

The best archived solution gets an extra weight bonus (elitism_factor=2.0). All weights are then normalized so they sum to 1. This means sampling is biased toward the best known solution while still exploring others.

✅ Strengths

  • Faster convergence than plain AS
  • Strong exploitation of best solutions

❌ Weaknesses

  • Risk of premature convergence if the best solution is a local optimum
🐜📊
Rank-Based Ant System (RBAS)
Better ants get louder voices
ACO ▼
Analogy

Imagine the archive is a ranked leaderboard. #1 gets k votes, #2 gets k-1 votes, ..., last place gets 1 vote. Sampling is proportional to votes. The leader has strong influence but everyone still contributes.

How it differs from AS

Weights follow a linear decay: w[i] = (pop_size - i) for rank i=0..pop_size-1. The best solution gets the most weight, linearly decreasing to the worst. This is a principled middle ground between the equal weights of AS and the extreme bias of EAS.

✅ Strengths

  • Smooth, principled balance between exploration and exploitation
  • No need to tune an elitism factor

❌ Weaknesses

  • The linear decay may not be optimal for all problem shapes
🐜⚖️
MAX-MIN Ant System (MMAS)
Trails can't get too strong or too weak
ACO ▼
Analogy

Same ant colony, but now there's a rule: no trail can ever become so dominant that it drowns out everything else, and no trail can become so faint it disappears. Pheromone levels are clamped between a minimum and maximum. If the colony gets stuck (no improvement for 20 rounds), the weaker solutions get replaced to inject fresh diversity.

What makes it special

  • Pheromone clamping: Weights are bounded between τ_min and τ_max. This prevents any single solution from monopolizing the search.
  • Stagnation restart: If no improvement occurs for 20 consecutive iterations, the bottom 50% of the archive is regenerated randomly. This prevents getting stuck.

✅ Strengths

  • Best of the four ACO variants for avoiding stagnation
  • Auto-restarts when stuck
  • Principled diversity maintenance

❌ Weaknesses

  • More complex — stagnation detection adds overhead
  • Restarts can waste progress if triggered too often

Group C — Evolutionary Algorithm

🧬
Genetic Algorithm (GA)
Survival of the fittest clusters
Evolutionary ▼
Analogy

Evolution in nature: a population of organisms (centroid configurations). The fittest survive. Two parents mate and produce offspring with traits from both. Sometimes a gene mutates randomly. Repeat for many generations. The population gradually evolves toward optimal solutions.

Step by step

  1. Initialize population: Create pop_size random centroid configurations. These are the "chromosomes" (the DNA = centroid values).
  2. Evaluate fitness: Compute WCSS for each individual. Lower WCSS = higher fitness.
  3. Elitism: The top 2 individuals automatically survive unchanged to the next generation.
  4. Tournament selection: To pick a parent, randomly draw 3 individuals and pick the best. Repeat to get a second parent. (Tournament size=3 means moderate selection pressure.)
  5. BLX-0.5 Crossover: With 80% probability, two parents "mate". For each centroid dimension, pick a value in the range [min(p1,p2) - 0.5×range, max(p1,p2) + 0.5×range]. The 0.5 extension lets offspring explore slightly beyond either parent.
  6. Gaussian Mutation: With 10% probability, add small random noise (std=0.1) to each centroid value.
  7. New generation: Replace old population with offspring. Go back to step 2.

Key parameters

ParameterWhat it does
crossover_rate=0.880% of pairs actually cross over; rest are copied unchanged.
mutation_rate=0.110% chance any given centroid value is mutated.
tournament_size=3Size of tournament for parent selection — higher = more pressure on best.
elitism_count=2Number of top individuals preserved unchanged each generation.

✅ Strengths

  • Excellent exploration — diverse population
  • Crossover combines good traits from multiple solutions
  • Well-understood theoretical properties

❌ Weaknesses

  • Can be slow to converge on precise solutions
  • Many hyperparameters to tune
  • Crossover can sometimes disrupt good solutions

Group D — Swarm Intelligence

🐦
Particle Swarm Optimization (PSO)
Birds flocking to food
Swarm ▼
Analogy

Imagine a flock of birds searching for food. Each bird remembers the best location it personally found, and the flock knows where the globally best food source is. Every bird adjusts its speed based on: (1) momentum (keep going the same way), (2) pull toward its own best spot, and (3) pull toward the flock's best spot. Over time the flock converges on the food.

Step by step

  1. Initialize swarm: Each of pop_size "particles" gets a random centroid configuration (position) and a random velocity.
  2. Evaluate: Compute WCSS for each particle. Record each particle's personal best (pbest) and the global best (gbest) across the entire swarm.
  3. Update velocity: For each particle: v = w×v + c1×r1×(pbest−x) + c2×r2×(gbest−x). Here w is inertia (decays from 0.9 to 0.4 over iterations), c1=c2=2.0, r1/r2 are random numbers. Velocity is clamped to prevent explosions.
  4. Update position: Move each particle: x = x + v. Clamp to data range.
  5. Update pbest/gbest: If a particle found a new personal best, update pbest. Update gbest if any particle beats it.
  6. Repeat.

Key parameters

ParameterWhat it does
w_init=0.9, w_end=0.4Inertia decays from 0.9 (explore) to 0.4 (exploit) — starts exploring, finishes fine-tuning.
c1=2.0Cognitive coefficient — strength of pull toward particle's own best.
c2=2.0Social coefficient — strength of pull toward the global best.

✅ Strengths

  • Fast convergence — social sharing of best solution speeds up all particles
  • Very few effective parameters
  • Works well in continuous spaces like centroid coordinates

❌ Weaknesses

  • Can collapse around a premature solution — all particles swarm to a local optimum
  • No crossover-style gene mixing

Group E — Hybrid Algorithm

🧬🐦
Hybrid GA-PSO
Birds that can also breed
Hybrid ▼
Analogy

The PSO swarm of birds, but every iteration the weakest 50% of birds don't just move toward the best food — they also mate and evolve. The offspring inherit traits from two parent birds. This injects genetic diversity into the swarm and helps it escape local optima that pure PSO would get stuck in.

Step by step — per iteration

  1. PSO phase: All particles update their velocity and position using the standard PSO equations (inertia + personal best pull + global best pull).
  2. GA phase (worst 50% only): Sort particles by current WCSS. For the bottom half:
    • Run tournament selection to pick two parents from the full population
    • Apply BLX-0.5 crossover to produce an offspring
    • Apply Gaussian mutation with probability 5%
    • Replace the weak particle with the offspring
  3. Update pbest/gbest: Check if any particle found a new best.
  4. Repeat.

Why hybrid?

PSO is fast at exploiting good regions but converges early. GA crossover reshuffles solutions and creates diversity. By applying GA only to the worst half, the algorithm keeps the good particles intact (exploitation) while rebuilding the weak ones (exploration).

✅ Strengths

  • Better exploration than pure PSO
  • Better convergence speed than pure GA
  • Often outperforms both parents on complex problems

❌ Weaknesses

  • More hyperparameters (both PSO and GA params)
  • Computationally heavier per iteration
  • Interaction between PSO and GA can sometimes cancel out benefits

6 — The 5 Performance Metrics

After each algorithm runs, its result is measured using five different "rulers". No single metric tells the whole story — use them together.

WCSS — Within-Cluster Sum of Squares ↓ lower = better

What it measures: How tightly packed the clusters are. Formally: for every data point, compute the squared distance to its centroid, then add them all up. The total is WCSS.

Analogy

Imagine each customer drives from their home to their assigned store (centroid). WCSS is the total fuel spent by all customers combined. Less fuel = tighter, more efficient clusters.

How to read it: There's no universal "good" value — it depends on the dataset. Compare it across algorithms on the same dataset. Lower is always better.

Limitation: WCSS always decreases as you add more clusters (k). Don't use it to decide the right number of clusters — use it only to compare algorithms at the same k.

How It's Calculated

WCSS = Σk Σx ∈ Ck ||x − μk||² k      = cluster index (1 to K) x      = a single data point (e.g. a customer's feature vector) Ck    = the set of all points assigned to cluster k μk    = centroid of cluster k (mean of all x in Ck) ||.||² = squared Euclidean distance (sum of squared differences per feature)

Worked Example — 3 points, 2 clusters

Setup: Points A=(1,1), B=(2,1), C=(8,8). Centroids: μ1=(1.5, 1), μ2=(8, 8).

Assign: A→C1 (dist² = (1−1.5)²+(1−1)² = 0.25),  B→C1 (dist² = (2−1.5)²+(1−1)² = 0.25),  C→C2 (dist² = 0).

Calculate: WCSS = 0.25 + 0.25 + 0 = 0.50

Meaning: A lower value means data points sit closer to their group's centre. Compare this number across algorithms at the same k — the one with the smallest WCSS found more compact groupings.

Low WCSS = compact clusters ✓ High WCSS = spread-out clusters ✗
Silhouette Score ↑ higher = better | range: −1 to +1

What it measures: For each data point, it compares (a) how close it is to other points in its own cluster vs (b) how close it is to the nearest other cluster. A high score means "I clearly belong here and I'm far from the next cluster".

Analogy

Each customer rates: "How well do I fit my assigned group vs how close am I to the next best group?" A score near +1 = perfect fit. Score near 0 = on the border. Score near −1 = probably in the wrong group.

How It's Calculated

s(i) = ( b(i) − a(i) ) / max( a(i), b(i) ) a(i) = mean distance from point i to all other points in its own cluster (cohesion) b(i) = mean distance from point i to all points in the nearest other cluster (separation) s(i) = silhouette score for a single point, range [−1, +1] Final score = mean of s(i) across all n data points

Worked Example — 1 point, 2 clusters

Setup: Point P is in Cluster 1. Mean distance to its cluster-mates: a(P) = 0.4. Mean distance to all points in the nearest other cluster: b(P) = 1.8.

Calculate: s(P) = (1.8 − 0.4) / max(0.4, 1.8) = 1.4 / 1.8 = 0.778

Negative case: If a(P) = 1.5 and b(P) = 0.6, then s(P) = (0.6 − 1.5) / 1.5 = −0.60 — P is closer to the other cluster than its own, suggesting a misassignment.

Meaning: Scores close to +1 mean tight, well-separated clusters. Scores near 0 mean points sit on cluster boundaries. Negative scores flag likely misclassified points.

+1.0 = perfectly separated clusters 0.0 = ambiguous membership -1.0 = misassigned points

Rule of thumb: >0.5 = good, 0.25–0.5 = fair, <0.25 = poor clustering. Real-world data rarely exceeds 0.6.

Davies-Bouldin Index ↓ lower = better | range: 0 to ∞

What it measures: For each cluster pair, it computes: (scatter within cluster A + scatter within cluster B) / distance between their centroids. The DB index is the average of the worst-case ratio for each cluster.

Analogy

For each store, find its most "dangerously similar" competitor store. DB measures the average "danger" across all stores. Lower = clusters are well-separated and internally compact.

How It's Calculated

DB = (1/K) Σk=1..K maxj≠k [ (σk + σj) / d(μk, μj) ] σk = average distance of points in cluster k to their centroid μk (scatter / spread) σj = same measure for cluster j d(μk, μj) = Euclidean distance between the centroids of clusters k and j maxj≠k = pick the worst (most similar) other cluster for each cluster k K          = total number of clusters

Worked Example — 2 clusters

Setup: σ1=0.5 (cluster 1 scatter), σ2=0.8 (cluster 2 scatter), d(μ1, μ2)=3.0.

Ratio for cluster 1: (0.5 + 0.8) / 3.0 = 0.433

DB (K=2, one pair each direction): DB = (1/2) × (0.433 + 0.433) = 0.433

Bad case: If the centroids were only 0.5 apart, the ratio becomes (0.5+0.8)/0.5 = 2.6 — clusters nearly overlap, DB shoots up.

Meaning: A high numerator (scattered clusters) OR a low denominator (close centroids) both push DB up (worse). Perfect, non-overlapping clusters give DB = 0.

0 = perfect separation (ideal) High values = clusters overlap or are spread out
Calinski-Harabasz Score ↑ higher = better | range: 0 to ∞

What it measures: The ratio of between-cluster variance to within-cluster variance. Think of it as a signal-to-noise ratio for clustering. High CH means clusters are well-separated (high signal) relative to how spread-out they are internally (low noise).

Analogy

How far apart are the different stores vs how spread-out are customers within each store's zone? High CH = stores are far from each other AND customers are close to their store.

How It's Calculated

CH = [ SSB / (K−1) ] ÷ [ SSW / (n−K) ] SSB = between-cluster scatter: Σk nk × ||μk − μglobal||² SSW = within-cluster scatter (= WCSS): Σk Σx∈Ck ||x − μk||² nk     = number of points in cluster k n        = total number of data points K        = number of clusters μglobal = mean of the entire dataset

Worked Example — K=2, n=6

Given: SSB=120 (clusters far from global mean), SSW=30 (points close to their centroids), K=2, n=6.

Calculate: CH = (120 / (2−1)) ÷ (30 / (6−2)) = 120 ÷ 7.5 = 16.0

Meaning: A ratio of 16 means the between-cluster spread is 16× larger than the within-cluster spread — clusters are well-separated relative to their internal scatter. Higher is better; there is no upper bound.

Note: The (K−1) and (n−K) denominators are degrees-of-freedom corrections, similar to how variance is computed with n−1 instead of n.

High values = dense, well-separated clusters Low values = overlapping or diffuse clusters

Note: Like WCSS, this tends to keep increasing as k increases — use it for comparing algorithms at fixed k, not for choosing k.

Accuracy ↑ higher = better | range: 0 to 1

What it measures: How well the discovered clusters match actual known labels (the Response column — whether a customer accepted a marketing campaign). Uses the Hungarian algorithm to find the best assignment of cluster IDs to real labels.

Analogy

After sorting customers into groups, we check: "If each group corresponds to one type of campaign responder, how many customers are correctly labelled?" The Hungarian algorithm finds the optimal way to match groups to labels to maximize correct assignments.

Important: This metric only works if the dataset has reference labels (the Response column). It is shown as N/A if no labels are available. Also note: clustering algorithms are not trained to predict labels — accuracy here just measures alignment, not the algorithm's true purpose.

How It's Calculated

Accuracy = maxassignment Σi 1[ cluster_label(i) = true_label(i) ] / n The optimal assignment is found by the Hungarian algorithm in 3 steps:   1. Build a confusion matrix C where C[k, j] = number of points in cluster k with true label j   2. Find the permutation of cluster IDs that maximises the diagonal sum of C   3. Accuracy = best diagonal sum / total points n The Hungarian algorithm solves step 2 in polynomial time (O(K³)) without trying all K! permutations.

Worked Example — 2 clusters, 6 points

Setup: True labels: [0,0,0,1,1,1]. Predicted clusters: [A,A,B,B,A,B].

Confusion matrix:

Label 0Label 1
Cluster A21
Cluster B12

Assignment 1 (A→0, B→1): correct = 2+2 = 4    Assignment 2 (A→1, B→0): correct = 1+1 = 2.

Best: Accuracy = 4/6 = 0.667. Hungarian picks Assignment 1 automatically.

When K > number of labels: e.g. 4 clusters but only 2 labels — some clusters get no matching label. The algorithm still finds the best partial alignment and accuracy is computed over all n points.

1.0 = perfect label alignment 0.5 = random chance (for 2 classes) Low values = clusters don't match the known groups

7 — How to Read the Results

Which metric should I trust most?

It depends on your goal:

  • If you want the most compact clusters: use WCSS as the primary metric. The algorithm with the lowest WCSS won.
  • If you want clusters that are meaningful and well-separated: use Silhouette. It's the most interpretable metric.
  • If you have ground-truth labels: use Accuracy. But remember — these algorithms weren't designed to predict labels, so don't be surprised if accuracy is low.
  • For a balanced view: look at all four unsupervised metrics (WCSS, Silhouette, DB, CH) together. A good solution should rank well on most of them.

What if metrics disagree?

This is very common! One algorithm might have the best WCSS but mediocre Silhouette. This can happen because:

  • WCSS rewards compact clusters but ignores separation between clusters
  • Silhouette considers both compactness and separation
  • Different metrics have different sensitivities to cluster shapes and sizes

When metrics disagree, look at the scatter plots. The visual structure of the clusters often tells you more than any single number.

Understanding the convergence plot

The Convergence tab shows how WCSS evolved over iterations. A steep drop early followed by flattening = the algorithm converged quickly. A slow, gradual descent = it kept improving throughout. A flat line = the algorithm is non-iterative (KNN-based) or got stuck.

Why do results change between runs?

Most algorithms are stochastic — they use random numbers. Different random seeds → different starting positions → potentially different final solutions. That's why multi-seed averaging is important: it shows you the typical performance, not just one lucky (or unlucky) run.

8 — What Is Multi-Seed Averaging?

When you set Number of Seeds to N > 1, each algorithm is run N times with seeds seed, seed+1, ..., seed+N-1. The reported metrics are the mean and standard deviation across all N runs.

Why does this matter?

A single run might be misleadingly good or bad due to random chance. For example, K-means with seed=42 might happen to start near an excellent solution, while seed=43 might start poorly. Averaging over 5 or 10 seeds gives a much more reliable picture of how each algorithm typically performs.

Analogy

Would you judge a darts player by a single throw? No — you'd watch 10 throws and compute their average score and consistency (standard deviation). Multi-seed averaging does the same for clustering algorithms.

How to interpret mean ± std

  • Low std (e.g., ±0.001): The algorithm is consistent — you get similar results regardless of the seed. Great for reproducibility.
  • High std (e.g., ±500): The algorithm is sensitive to initialization — sometimes great, sometimes terrible. You need multi-seed averaging to get a reliable estimate.

9 — Glossary

Algorithm
A step-by-step procedure for solving a problem. Like a recipe for a computer.
Archive
In ACO algorithms, a stored collection of good solutions that new solutions are sampled from.
Centroid
The "centre of gravity" of a cluster — the average position of all points in that cluster.
Chromosome
In GA, the encoding of a solution as a list of numbers (like DNA). Here it's the flat list of all centroid coordinates.
Clustering
Automatically dividing data into groups so similar items are together and dissimilar items are apart.
Convergence
When an algorithm stops improving significantly. The convergence curve shows WCSS over time.
Crossover (GA)
Combining genes from two parent solutions to create a child solution. Like biological reproduction.
Davies-Bouldin Index
Metric measuring cluster quality as the ratio of within-cluster scatter to between-cluster separation. Lower = better.
Elitism
Preserving the best solutions unchanged into the next generation, preventing accidentally losing good results.
Euclidean Distance
Straight-line distance between two points. Used by all algorithms here to measure how close a customer is to a centroid.
Exploration vs Exploitation
The fundamental tradeoff: explore new regions (risk) vs exploit what you know is good (safety). Good algorithms balance both.
Fitness Function
The function being optimized. Here it's WCSS — all algorithms try to minimize it.
Global Best (gbest)
In PSO, the best solution found by any particle across all iterations.
Hungarian Algorithm
A mathematical method for optimally matching one set of items to another. Used here to align predicted cluster IDs with true labels for accuracy calculation.
Inertia (PSO)
The tendency of a particle to keep moving in the same direction. Decays over time so particles slow down and fine-tune.
k (number of clusters)
The one parameter all algorithms share — how many groups to divide the data into. Must be chosen by the user.
Local Optimum
A solution that's better than all its neighbours but not the global best. K-means is famous for getting stuck here.
Lloyd's Iteration
The core K-means update: assign each point to the nearest centroid, then move centroids to the mean of their assigned points. Repeat until stable.
Metaheuristic
A high-level optimization strategy (like "evolve" or "follow pheromones") that guides simpler algorithms. Not algorithm-specific — can be applied to many problem types.
Mutation (GA)
Randomly changing a small part of a solution. Provides diversity and helps explore new areas of the solution space.
PCA (Principal Component Analysis)
A technique to reduce many features down to 2 for visualization. Preserves as much variation as possible so you can see cluster structure in 2D.
Personal Best (pbest)
In PSO, the best position a particle itself has visited during the entire search.
Population
In population-based algorithms (GA, PSO, ICA, etc.), a set of multiple solutions maintained simultaneously. Diversity in the population helps avoid local optima.
Random Seed
A number that initializes the random number generator. Same seed → same random numbers → same result. Different seed → different random path through the search.
Silhouette Score
Measures how well each point fits its cluster vs the next-closest cluster. Range −1 to +1; higher is better.
Standard Deviation (std)
A measure of how spread out results are. Low std = consistent results. High std = highly variable results across runs.
StandardScaler
Preprocessing step that rescales each feature to have mean=0 and std=1. Prevents features with large ranges (like income) from dominating features with small ranges (like age).
Stochastic
Random. A stochastic algorithm uses randomness and may give different results each run with different seeds.
Tournament Selection (GA)
Parent selection method: randomly draw 3 candidates, pick the best one. Balances pressure to pick good parents with some randomness.
WCSS
Within-Cluster Sum of Squares — the primary optimization target. Sum of squared distances from each point to its centroid. Lower = better.

Pseudocode & Mathematical Reference

This section provides formal pseudocode and mathematical notation for the core operators of each algorithm family. All algorithms minimise the same objective — WCSS — using different search strategies.

Shared Fitness Function: WCSS minimised by all algorithms

Every candidate solution M = [μ₁ ‖ … ‖ μk] ∈ ℝk·d is evaluated as:

f(M) = WCSS(X, M) = Σᵢ minj∈{1..k} ‖xᵢ − μⱼ‖²

After every iteration, a Lloyd's refinement step recomputes centroids analytically:

labels[i] = argmin_j ‖x_i − μ_j‖²
μ_j       = mean({ x_i : labels[i] == j })   // if empty: reinit to random x_i
🧬 Genetic Algorithm — Operators & Math Selection · Crossover · Mutation · Elitism ▾

Tournament Selection

TournamentSelect(P, T):
    S ← sample T individuals uniformly from P
    return argmin_{p ∈ S} f(p)      // fittest wins

With T=3, weak individuals survive ~(1−1/T)^T ≈ 30% of the time, maintaining diversity.

BLX-α Crossover (α = 0.5)

BLX-α(p1, p2):
    for each gene g:
        lo_g = min(p1_g, p2_g) − α · |p1_g − p2_g|
        hi_g = max(p1_g, p2_g) + α · |p1_g − p2_g|
        child_g ~ Uniform(lo_g, hi_g)
    return child

α=0.5 extends 50% beyond parent range — offspring can explore outside the parent convex hull.

Gaussian Mutation

Mutate(ind, p_m=0.1, σ=0.1):
    for each gene g:
        if Uniform(0,1) < p_m:
            ind_g ← ind_g + N(0, σ)
    return clip(ind, lb, ub)

Full GA Loop

Initialise P: pop_size random centroid sets drawn from X
Evaluate f(p) = WCSS for all p ∈ P

for t = 1 to max_iter:
    sort P by f(p) ascending
    new_P ← P[0 : elitism_count]           // preserve elite
    while |new_P| < pop_size:
        p1 ← TournamentSelect(P, 3)
        p2 ← TournamentSelect(P, 3)
        child ← BLX-α(p1, p2)  if rand() < 0.8  else p1
        child ← Mutate(child)
        new_P.append(child)
    P ← new_P
    Evaluate f(p) for all p ∈ P
    record best_WCSS[t] ← min f(p)
🐦 Particle Swarm Optimisation — Velocity Update Inertia · Cognitive · Social · Clamping ▾

Inertia Weight Decay

w(t) = w_init − (w_init − w_end) × t / (T − 1)
     = 0.9 → 0.4  (linearly decreasing)

High early inertia drives broad global search; low late inertia enables fine-grained local exploitation.

Velocity & Position Update

// Per particle i, per dimension d:
r1, r2 ~ Uniform(0, 1)

v_i(t+1) = w(t) · v_i(t)
          + c1 · r1 · (pbest_i − x_i(t))    // cognitive pull
          + c2 · r2 · (gbest  − x_i(t))     // social pull

v_i(t+1) = clip(v_i(t+1), −v_max, +v_max)  // v_max = 0.2(ub−lb)
x_i(t+1) = clip(x_i(t) + v_i(t+1), lb, ub)

if f(x_i(t+1)) < f(pbest_i):
    pbest_i ← x_i(t+1)
gbest ← argmin_i f(pbest_i)
⚡ Hybrid GA-PSO — Per-Iteration Scheme PSO exploitation + GA diversity on worst particles ▾ 
for t = 1 to max_iter:
    // Phase 1: PSO on ALL particles
    for each particle i:
        update velocity and position (standard PSO, c1=c2=1.5)
        update pbest_i if improved

    // Phase 2: GA on WORST 50% by WCSS
    sorted_idx ← argsort(f(x_i)) descending
    worst_half ← sorted_idx[0 : pop_size//2]
    for each pair (i, j) in worst_half:
        child ← BLX-α(x_i, x_j)
        child ← Mutate(child, p_m=0.05)
        if f(child) < f(x_i):
            x_i ← child  // replace if improved

PSO ensures exploitation of the current best; GA operators rejuvenate stagnated particles without disturbing the elite.

🐜 ACO (ACOR) — Archive Sampling Pheromone weights · Gaussian sampling · MMAS clamping ▾

Archive Weight Formulas

VariantWeight w_l for archive entry l (rank 0=best)
Ant Systemw_l = 1/K (uniform)
Elitist ASw_l = 1/K + elitism_factor × 𝟙[l=0]
Rank-Based ASw_l = (K−l) / Σⱼ(K−j)
MMASw_l = Gauss(l; 0, τ_max), clamped to [τ_min, τ_max]

Ant Sampling (per dimension d)

Select archive entry l with prob ∝ w_l

σ_d = ξ × (1/(K−1)) × Σ_j |archive[l,d] − archive[j,d]|
                                                    // ξ=0.85 controls spread

x_new_d ~ N( archive[l,d], σ_d )

MMAS Stagnation Restart

if best_WCSS unchanged for stagnation_limit=20 iterations:
    bottom_half ← archive[K//2 : K]
    reinitialise bottom_half with random data points from X
👑 Imperialist Competitive Algorithm — Core Operators Assimilation · Revolution · Competition · Collapse ▾ 
// Assimilation: colony moves toward its imperialist
β       ~ Uniform(0, assimilation_coeff=2.0)
θ       ~ Uniform(−π/4, π/4)         // angular noise
colony ← colony + β × R(θ) × (imperialist − colony)

// Revolution: random reinitialisation at rate r
if Uniform(0,1) < revolution_rate=0.1:
    colony ← random centroid set from X

// Role swap if colony beats imperialist
if WCSS(colony) < WCSS(imperialist):
    swap(colony, imperialist)

// Empire power for competition
power_e = WCSS(imperialist_e) + 0.1 × mean(WCSS(colonies_e))

// Competition: weakest colony of weakest empire → strongest empire
weakest_empire ← argmax_e power_e
colony_to_move ← argmax_{c ∈ weakest_empire} WCSS(c)
strongest_empire ← argmin_e power_e
move colony_to_move to strongest_empire

// Collapse: if empire has no colonies, disband
if |colonies_e| == 0:
    distribute imperialist_e as colony to remaining empires
💥 Extremal Optimisation & τ-EO Component fitness · Rank-based selection · Power-law ▾ 
// EO: always replace worst component
for t = 1 to max_iter:
    compute per-centroid WCSS contribution: f_j = Σ_{xᵢ assigned to j} ‖xᵢ − μⱼ‖²
    worst ← argmax_j f_j
    μ_worst ← random data point from X
    refine centroids (Lloyd's step)
    record best_WCSS[t]

// τ-EO: rank-biased probabilistic replacement
for t = 1 to max_iter:
    rank centroids by f_j ascending (rank 1 = worst)
    P(replace centroid j) ∝ rank(j)^(−τ),  τ=1.5
    select j to replace according to P
    μ_j ← random data point from X
    refine centroids (Lloyd's step)

τ-EO's power-law distribution means even good components are occasionally replaced (with low probability), preventing the deterministic pathology of plain EO where the second-worst component is never touched.

🎵 Harmony Search — Memory & Pitch Adjustment HMCR · PAR · Bandwidth decay ▾ 
// Bandwidth annealing schedule
bw(t) = bw_init × exp(−t × ln(bw_init / bw_final) / T)
      = 0.02 → 1×10⁻⁴

for t = 1 to max_iter:
    for each gene dimension d:
        if Uniform(0,1) < HMCR=0.9:                   // memory consideration
            x_new_d ← HM[random member, d]
            if Uniform(0,1) < PAR=0.3:                // pitch adjustment
                x_new_d ← x_new_d + Uniform(−bw, +bw)
        else:
            x_new_d ← Uniform(lb_d, ub_d)             // random exploration

    if f(x_new) < f(worst in HM):
        replace worst in HM with x_new

HMCR=0.9 means 90% of the time a new harmony reuses memory (exploitation); 10% explores randomly (diversity). PAR controls local refinement within memory-drawn values.

Evaluation Metric Formulas

Silhouette Score
a(i) = mean intra-cluster distance for point i
b(i) = min mean distance to any other cluster
s(i) = (b(i) − a(i)) / max(a(i), b(i))
Silhouette = mean s(i)  ∈ [−1, +1]   // higher = better
Davies-Bouldin Index
s_i = mean(‖x − μᵢ‖ for x in cluster i)   // intra-cluster scatter
d_ij = ‖μᵢ − μⱼ‖                           // inter-centroid distance
DB = (1/k) Σᵢ max_{j≠i} (sᵢ + sⱼ) / d_ij  // lower = better
Calinski-Harabasz Index
SS_B = Σⱼ nⱼ · ‖μⱼ − μ‖²   // between-cluster scatter (μ = grand mean)
SS_W = Σⱼ Σ_{i in j} ‖xᵢ − μⱼ‖²   // within-cluster scatter (= WCSS)
CH   = [SS_B / (k−1)] / [SS_W / (n−k)]   // higher = better
Clustering Accuracy (Hungarian Algorithm)
Build confusion matrix C where C[i,j] = |{x : y_true=i, y_pred=j}|
Find optimal assignment π* = Hungarian(C)   // maximises Σ C[i, π*(i)]
Accuracy = Σᵢ C[i, π*(i)] / n              // higher = better

The Hungarian algorithm resolves the label permutation ambiguity inherent in clustering (cluster IDs have no canonical ordering).

✓ EARNED. The Hybrid GA-PSO combines PSO (swarm intelligence) and GA (evolutionary computation) within a single per-iteration loop. PSO exploits the current best region for all particles; GA diversifies the worst half. This is not a sequential combination (run PSO then GA) but a cooperative coupling — both paradigms act every iteration on the same population. Reference: Kao & Zahara (2008).
